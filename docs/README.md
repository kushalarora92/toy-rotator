# Documentation

This directory contains project documentation organized by feature.

## Structure

```
docs/
├── {number}.{feature-name}/           # Feature group (numbered)
│   ├── {YYYY-MM-DD}_{description}.md  # Feature documentation
│   └── README.md                      # Feature index
└── README.md                          # This file
```

## Conventions

- **Folders**: `{number}.{feature-name}/` — lowercase, hyphenated, numbered by priority
- **Files**: `{YYYY-MM-DD}_{descriptive-name}.md` — date prefix for chronology
- Group related documents together (setup, implementation, testing, troubleshooting)
- Keep documentation up-to-date when making code changes
